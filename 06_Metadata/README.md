# ⚙️ 元数据

仓库配置、文档和组织工具。

## 目的

元数据文件夹包含:
- 关于仓库的文档
- 笔记创建的模板
- 参考指南和操作指南
- 代理配置
- 工作流程文档

## 结构

```
06_Metadata/
├── Reference/         # 指南和文档
├── Templates/        # 笔记模板
├── Agents/          # Claude Code 代理配置
├── Workflows/       # 记录的流程
└── Archive/        # 旧配置
```

## 这里的内容

### Reference
- 此仓库的文档
- Claude Code 提示库
- 风格指南
- 工作流程文档
- 学习资源

### Templates
- 项目模板
- 每日笔记模板
- 会议模板
- 研究模板
- 审查模板

### Agents
- 思考伙伴说明
- 研究助手配置
- 编辑代理设置
- 自定义代理定义

### Workflows
- 每周审查流程
- 项目完成检查清单
- 收件箱处理指南
- 归档程序

## 使用模板

### 手动
1. 复制模板内容
2. 创建新笔记
3. 粘贴并填写

### 使用 Claude Code
```
使用项目模板创建一个新项目。
将其命名为 [项目名称] 并放在 01_Projects 中。
```

## 创建自定义代理

将代理说明保存为 markdown 文件:

```markdown
# 代理: [名称]

你是一个 [角色描述]。

## 核心行为
- 行为 1
- 行为 2

## 工作流程
1. 步骤 1
2. 步骤 2

## 约束
- 不要做 X
- 始终做 Y
```

然后在 Claude Code 中引用:
```
使用 06_Metadata/Agents/[agent].md 中的说明
并帮助我 [任务]。
```

## Claude Code 提示

### 模板使用
```
显示 06_Metadata/Templates 中的可用模板。
使用适当的模板创建一个新的 [类型] 笔记。
```

### 文档
```
检查 06_Metadata/Reference 中关于 [主题] 的文档。
根据我们刚刚学到的内容更新指南。
```

### 工作流程执行
```
从 06_Metadata/Workflows 运行每周审查工作流程。
引导我完成每个步骤。
```

## 维护

### 定期更新
- 根据使用情况更新模板
- 记录新出现的工作流程
- 归档过时的配置
- 保持参考文档最新

### 版本控制
- 跟踪工作流程的更改
- 记录为什么进行更改
- 保留旧版本的存档
- 为主要更新添加日期

## 最佳实践

- **边做边记录** - 在新鲜时捕获工作流程
- **迭代模板** - 根据使用情况改进
- **分享配置** - 对你有效的东西可能帮助他人
- **保持简单** - 复杂的系统会崩溃
- **一切加日期** - 背景很重要

## 记住

元数据是你仓库的操作系统。好的元数据意味着一致的结构、可重复的工作流程和可扩展的增长。这里你不仅记录你知道什么,还记录你如何工作。

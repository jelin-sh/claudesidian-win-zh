# 🎯 项目

具有具体结果和截止日期的活跃倡议。

## 目的

项目是**有时间限制的努力**,具有:
- 明确的目标和可交付成果
- 定义的开始和结束日期
- 具体的成功标准
- 可衡量的结果

## 什么属于这里

### 好的项目
- "2024 年第一季度营销策略" ✓
- "网站重新设计" ✓
- "书籍写作 - AI 与创造力" ✓
- "会议演讲 - BRXND NYC" ✓

### 不是项目(这些放在领域)
- "健康" ✗ (持续进行,没有结束日期)
- "学习" ✗ (持续活动)
- "团队管理" ✗ (持续职责)

## 项目结构

每个项目应该有:
```
项目名称/
├── README.md           # 项目概览和状态
├── Research/          # 背景材料
├── Daily_Progress/    # 工作会话日志
├── Drafts/           # 进行中的工作
├── Resources/        # 支持文档
└── Output/          # 最终可交付成果
```

## Claude Code 工作流程

### 启动项目
```
在 01_Projects 中创建一个名为[名称]的新项目。
使用标准文件夹结构设置它。
我处于思考模式 - 帮助我定义目标。
```

### 项目研究
```
我正在处理[项目名称]。
搜索我的仓库中相关的现有笔记。
与我的其他工作有什么联系?
```

### 每日进展
```
在 [project]/Daily_Progress 中为今天创建一个进展笔记。
这是我完成的内容:[摘要]
出现的问题:[列表]
```

### 项目状态
```
审查 [项目文件夹] 中的所有笔记。
当前状态如何?
到目前为止的关键洞察是什么?
还有什么需要完成?
```

### 项目完成
```
项目 [名称] 已完成。
创建一个回顾,包括:
- 目标与结果
- 关键学习
- 下次不同的做法
然后帮助我正确归档它。
```

## 项目生命周期

1. **启动**: 定义目标、成功标准
2. **研究**: 收集相关信息
3. **开发**: 每日进展、迭代工作
4. **审查**: 定期状态检查
5. **完成**: 最终输出、回顾
6. **归档**: 移至 04_Archive 并附带摘要

## 成功提示

- **从明确的目标开始** - 模糊的项目永远不会结束
- **使用模板** - 一致性有助于 Claude Code 帮助你
- **每天记录进展** - 即使"没有进展"也值得注意
- **每周审查** - 保持项目正轨
- **完成或终止** - 不要让项目拖延

## 何时归档

将项目移至 `04_Archive/` 当:
- 所有目标已完成
- 项目已取消
- 已不活跃 30 天以上
- 已转变为领域(持续进行)

归档前始终创建一个摘要笔记!

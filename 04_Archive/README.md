# 🗄️ 归档

为未来参考保存的非活跃项目。

## 目的

归档存储:

- 已完成的项目及其输出
- 不再维护的非活跃领域
- 用于历史参考的旧笔记
- 已弃用的资源
- 过去的实验和迭代

## 什么放在这里

### 来自项目

- 已完成的项目和最终可交付成果
- 已取消的项目和经验教训
- 不活跃 30 天以上的项目

### 来自领域

- 与你生活不再相关的领域
- 你已移交的职责
- 你不再担任的角色

### 来自资源

- 过时信息(但历史上有趣)
- 被取代的框架或方法
- 演变想法的旧版本

### 来自收件箱

- 已处理不再需要的项目
- 旧的每日笔记(提取后)
- 没有持久价值的随机捕获

## 组织

```
04_Archive/
├── Projects_2024/     # 按年度完成的项目
├── Projects_2023/
├── Old_Areas/        # 已停止的领域
├── Daily_Notes/      # 旧的每日捕获
├── Ideas/           # 没有发展的想法
└── Miscellaneous/   # 其他所有内容
```

## 归档流程

### 归档项目之前

1. 创建完成摘要
2. 将可重用的洞察提取到 Resources
3. 记录经验教训
4. 更新任何相关的 Areas
5. 保持结构完整地移动整个文件夹

### 完成摘要示例

```markdown
# 项目: [名称] - 完成摘要

**持续时间**: 开始日期 - 结束日期
**状态**: 已完成/已取消/已暂停

## 目标

- 原始目标 1 ✓
- 原始目标 2 ✓
- 原始目标 3 ✗

## 关键结果

- 交付了什么
- 产生了什么影响
- 创造了什么价值

## 经验教训

- 哪些效果很好
- 哪些不起作用
- 下次不同的做法

## 可重用资产

- 创建的模板: [[链接]]
- 开发的流程: [[链接]]
- 获得的洞察: [[链接]]

## 相关笔记

- 继续于: [[领域名称]]
- 另见: [[相关项目]]
```

## 归档理念

### 不是墓地

- 归档保留机构记忆
- 旧项目为新项目提供信息
- 模式随时间出现
- 想法可以复活

### 不是垃圾场

- 有意识地归档
- 保持一定的组织
- 保持摘要可访问
- 删除真正的垃圾

## Claude Code 提示

### 历史分析
```
查看我的已归档项目。
我倾向于开始但不完成什么类型的事情?
什么主题反复出现?
```

### 知识挖掘
```
在归档中搜索对 [概念] 的任何提及。
我的想法如何演变?
```

### 模式识别
```
分析我的项目完成率。
什么因素与成功相关?
什么模式预测失败?
```

## 维护

### 每季度

- 审查最近的添加
- 确保摘要存在
- 检查复活候选者

### 每年

- 重大归档清理
- 删除真正死掉的内容
- 提取任何错过的洞察
- 如需要重新组织

## 提示

- **一切加日期** - 未来的你会感谢你
- **始终总结** - 背景很快淡化
- **自由链接** - 连接在归档中存活
- **经常搜索** - 归档意味着要被使用
- **无畏删除** - 不是所有内容都需要保留

## 记住

归档是你的机构记忆。它不是关于保留所有内容,而是关于保留可能为未来工作提供信息的内容。使用 Claude Code 帮助你看到跨时间的模式并从经验中提取智慧。

# 收件箱处理器

帮助按照 PARA 方法组织和处理 00_Inbox 文件夹中的项目。

## 任务

审查 `00_Inbox/` 中的所有笔记并帮助分类:

1. **扫描收件箱**
   - 列出当前 00_Inbox 中的所有文件
   - 排除 README.md 和 Welcome.md

2. **分析每个项目**
   - 阅读内容
   - 识别笔记类型
   - 建议合适的目标位置

3. **分类规则**
   - **→ 01_Projects**: 有截止日期、具体成果
   - **→ 02_Areas**: 持续责任、无结束日期
   - **→ 03_Resources**: 参考资料、知识
   - **→ 04_Archive**: 旧/已完成、不再活跃
   - **→ 删除**: 无价值、重复或临时

4. **建议操作**

   ```
   文件: [文件名]
   类型: [检测到的类型]
   目标位置: [建议的文件夹]
   原因: [为何这样分类]
   关联: [与现有笔记的联系]
   ```

5. **识别模式**
   - 多个笔记中的共同主题
   - 可以合并的笔记
   - 项目之间缺失的联系

## 输出格式

提供清晰的行动计划:

1. 要移动的项目(含目标位置)
2. 要合并或链接的项目
3. 要删除的项目
4. 需要更多上下文的项目

## 记住

- 有些项目确实属于收件箱(每日笔记、快速捕获)
- 不要过度组织 - 有时"足够好"就是完美
- 寻找连接想法的机会,而不仅仅是归档
